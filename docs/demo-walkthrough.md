# Demo 演示脚本

## 目标

用 2-3 分钟让评委看到：这不是一个单点推荐页面，而是一套从用户意图到会员运营的咖啡规划 Agent。

## 演示步骤

1. 打开 `http://localhost:5173/`。
2. 介绍产品名：小鹿 CoffeePlan，从一句需求生成可下单、可复购的咖啡计划。
3. 左侧选择“会议前提神”和会员“陈晨”。
4. 中间输入框保留默认需求：

```text
下午开会前想喝点提神，但别太甜，最好别排队。
```

5. 点击发送或直接看本轮规划结果：
   - 主意图：提神醒脑。
   - 推荐：生椰拿铁。
   - 到手：¥16。
   - 稳妥度：91%。
6. 右侧“方案”页说明：
   - 为什么是这杯。
   - 咖啡规划时间线：识别意图、匹配画像、组合优惠、规划门店、回写复购。
7. 点击“画像”页：
   - 展示会员状态、最近下单、周均频次、价格敏感度。
   - 展示偏好、避忌、券包。
8. 点击“确认这杯”：
   - 右侧自动进入“执行”页。
   - 展示模拟执行链路和真实支付二次确认说明。
9. 点击“模拟支付完成”：
   - 右侧自动进入“复购”页。
   - 展示复购话术、接受率、推荐点击提升、唤醒提升、复购周期缩短。
10. 切换到“小周 / 沉睡唤醒”：
   - 说明同一套 Agent 可服务沉睡会员唤醒，推荐、券和话术会变化。

## 讲解话术

“瑞幸已有 Lucky AI 可以帮用户下单，所以我的方案不重复做点单助手，而是把 Agent 变成流量运营层。用户不管从系统助手、社交搜索还是小程序入口进来，只要说一句自然语言需求，小鹿就会把意图识别、会员画像、券包、门店履约、支付确认和复购运营串起来。每一步都可解释、可兜底、可回写，最后沉淀为会员关系资产。”

## 注意事项

- Demo 使用合成数据。
- 不会调用真实瑞幸接口。
- 不会发起真实支付。
- 当前重点是证明产品链路和 Agent 工作方式，后续可接入真实会员、券包、门店和下单工具。
