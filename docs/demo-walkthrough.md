# Demo 演示脚本

## 目标

用 3 分钟让评委看到：小鹿 CoffeePlan 不是一个单点推荐页面，而是一套参考小满式双端体验、带真实商品视觉和执行护栏的咖啡 Agent 工作台。

## 演示前准备

```bash
npm install
npm run dev
```

打开：

```text
http://localhost:5173/luckin-loop-agent/
```

## 3 分钟主线

### 0:00 - 0:30 定位

讲解：

> 瑞幸已有 App/小程序和 Lucky AI 点单，小鹿不重复做客服，而是做“意图增长智能体”。用户从手机助手、系统 Agent 或社交搜索说一句需求，小鹿把它转成一杯可解释、可执行、可回写的咖啡计划。

屏幕动作：

1. 展示左侧 FlowNav 的“三幕咖啡旅程”。
2. 指出网页/手机切换、场景脚本、会员分身和能力挂载。

### 0:30 - 1:10 桌面工作台

屏幕动作：

1. 保持默认会员“陈晨 / 会议前提神”。
2. 看“今日任务”：生椰拿铁、官方商品图、模拟到手价、取餐 ETA、Agent 置信。
3. 切到“Agent 对话”：展示用户需求、Agent 推荐解释、执行前确认。
4. 切到“推荐策略”：展示候选召回评分、券包策略、兜底策略、订单路径。

讲解：

> 它不是只给一杯热门咖啡，而是把意图、偏好、避忌、门店、券包和信任风险联合排序。每个推荐都能解释为什么，以及哪里是 mock、哪里需要官方授权。

### 1:10 - 1:50 手机聊天体验

屏幕动作：

1. 点击左侧“手机”或把浏览器宽度缩到手机。
2. 展示手机端首屏：小鹿 Agent 对话、用户需求、定位卡、商品方案卡、底部输入。
3. 点击“拒绝定位”，展示“常购门店兜底”。

讲解：

> 手机端不是桌面后台缩窄，而是像小满一样先聊天。定位不是后台偷读，必须用户点击；拒绝后可以走最近订单/常购门店兜底。

### 1:50 - 2:30 自动执行

屏幕动作：

1. 点击“执行点单”。
2. 展示手机大窗自动走瑞幸式点单流程。
3. 展示底部链路弹窗：Step、Observation、Action。
4. 等流程推进或点击暂停，强调支付前停止。

讲解：

> 自动执行模块参考 GUI-Agent 的 Observe → Think → Act，但只用于执行模块。它展示本地模拟操作，不启动真实瑞幸 App；所有路径停在支付前，不创建订单、不扣券、不扣款。

### 2:30 - 3:00 数据接入与边界

屏幕动作：

1. 回到网页视图，切到“数据接入”。
2. 展示公开来源：Luckin Coffee US 官方菜单。
3. 展示 Adapter 蓝图：官方菜单图片、定位授权、会员权益、下单支付、门店履约。

讲解：

> 商品图片来自公开官方菜单，但中国区价格、库存、会员、券和支付都没有在 demo 中冒充真实接口。真实落地时，每个 adapter 都有 reads/writes、权限、guardrail 和 UI disclosure。

## 推荐补充演示

### 沉睡唤醒

切换到“小周 / 沉睡唤醒”：

- 商品会偏向低风险新品。
- 券解释强调即将过期和简单规则。
- 增长回写会变成低打扰唤醒。

### 低糖轻负担

切换到“Ada / 低糖轻负担”：

- 推荐解释会强调低糖、轻负担和营养透明。
- 兜底会检查低糖标签是否清楚。

### 老样子快取

切换到“老李 / 老样子快取”：

- 推荐更倾向最近订单和出杯稳定。
- 高频用户避免强推新品，减少打扰。

## 关键卖点话术

> 小鹿 CoffeePlan 的价值不在“AI 会聊天”，而在“AI 能把咖啡消费这件小事办稳”。它懂意图、懂会员、懂券、懂门店，也懂哪里不能越界。用户看到的是一杯可信的咖啡计划，运营侧沉淀的是可回写的会员关系资产。

## 注意事项

- Demo 使用本地合成会员、券包、门店和价格。
- 商品图片来自公开官方菜单页面，但不代表瑞幸中国区实时菜单。
- 浏览器定位只在用户点击后触发，不保存坐标。
- 不调用真实瑞幸接口，不发起真实支付。
- 自动执行只在本地 UI 模拟，并且必须停在支付前。
