# 瑞幸命题报名表文案

## 开题报告 part 1：命题前置分析与洞察

AI Agent 时代，用户可能先向手机助手、系统级 Agent、车机或社交搜索表达“今天想喝什么”，而不是主动打开品牌 App。瑞幸已有高频 App/小程序交易和 Lucky AI 点单基础，机会不应停在“聊天下单”，而是把外部入口里的模糊需求沉淀为可解释、可运营、可复购的会员关系资产。咖啡决策高度场景化，时间、口味、健康偏好、券包、门店距离、排队和取餐稳定性都会影响转化；如果 Agent 只返回热门菜单，会失去品牌信任和运营深度。

## 开题报告 part 2：整体解决方案设计

方案命名为“小鹿 CoffeePlan”，定位为瑞幸意图增长智能体：用户只说一句“下午开会前想提神，别太甜，最好别排队”，Agent 就生成一杯可解释、可执行、可回写的咖啡计划。系统分为五层：1）Intent Router 识别提神、低糖、快取、尝新、优惠等即时意图；2）Member Memory 结合合成会员画像、历史订单、偏好、避忌、券包和信任风险；3）Menu Store Tools 联合商品、官方公开商品图、门店距离、排队和 ETA 做候选召回；4）Coupon Optimizer 输出到手价、券使用理由和放弃风险；5）Growth Loop 将接受、拒绝定位、放弃执行、支付前停止等反馈回写为复购策略。

Demo 参考小满式双端体验：网页端是三栏工作台，左侧承载任务、会话、生活、记忆、商家和接入入口，中间是大的聊天/引导窗口，右侧是方案、画像、执行、数据画布；手机端默认先进入聊天流，不是桌面缩窄版。用户提交后页面显示用户消息，并先给出几个候选方向让用户选择，不直接固定推荐。只有点击“执行点单”后，才进入手机点单模拟，大窗展示瑞幸式下单过程，底部弹窗显示 Step、Observation、Action 链路。自动执行模块参考 GUI-Agent 的观察、推理、动作和护栏，但只用于执行部分，并严格停在支付前，不创建真实订单、不扣券、不扣款。

当前商品图片引用 Luckin Coffee US 官方公开菜单；中文价格、门店 ETA、库存、会员、券包均为 demo mock。真实落地时通过官方授权 adapter 接菜单、定位、会员权益、门店履约和支付链路，并在 UI 中明确数据来源、权限和边界。预期价值假设包括推荐接受率提升 10%-15%、沉睡会员唤醒提升 6%-11%、用户决策时长下降和履约放弃率下降，需通过灰度实验验证。
